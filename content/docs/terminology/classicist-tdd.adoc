_Классическая школа Test-Driven Development — подход к TDD, в котором тесты пишутся через публичное API и проверяют наблюдаемое поведение системы (в отличие от «лондонской школы», где акцент делается на проверке взаимодействий и использовании моков)._

* https://martinfowler.com/articles/mocksArentStubs.html[Mocks Aren't Stubs]
* https://martinfowler.com/articles/is-tdd-dead/[Is TDD Dead?]
